\chapter{总结与展望}\label{chap:conclusion}

\section{工作总结}
本文围绕 QuickBoard 在线协作白板系统的设计与实现开展研究与工程实践。系统以“无需注册、秒级创建/加入房间”为核心体验，基于 Vue3 与 Fabric 实现可编辑画布与基础绘图工具；基于 Node.js 与 Socket.IO 实现房间管理与实时协作通信，并采用服务端权威裁决与增量同步策略提升一致性体验；同时接入独立 OCR 服务实现数学公式识别，并在前端以 KaTeX 渲染与可编辑流程实现“识别—确认—插入—编辑”的闭环体验。通过单元测试、构建验证与典型场景测试，系统在可用性与稳定性方面达到预期目标，能够满足轻量协作白板的核心需求。

\section{后续展望}
后续可从以下方向进一步完善系统：
\begin{itemize}
  \item \textbf{更强一致性与离线能力}：在保持可用性的前提下，引入更细粒度的冲突合并或 CRDT 方案，并支持离线编辑与回放。
  \item \textbf{持久化与权限}：增加房间内容持久化存储与访问控制，支持更长期的协作与复用。
  \item \textbf{识别与编辑体验提升}：优化 OCR 预处理与多模型融合策略，并增强公式编辑器的交互与错误提示。
  \item \textbf{性能与跨端适配}：针对多对象与移动端场景进一步优化渲染与交互性能，提升跨端一致体验。
\end{itemize}
